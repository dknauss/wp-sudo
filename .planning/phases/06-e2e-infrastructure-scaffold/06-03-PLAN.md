  <action>
Create `.github/workflows/e2e.yml` as a standalone workflow file. Do NOT modify `phpunit.yml`.

```yaml
name: E2E Tests

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  e2e-tests:
    name: "E2E Tests (Playwright)"
    runs-on: ubuntu-24.04
    timeout-minutes: 30

    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version-file: '.nvmrc'
          cache: 'npm'

      - name: Install npm dependencies
        run: npm ci

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium

      - name: Start WordPress environment
        run: npx wp-env start

      - name: Wait for WordPress to be ready
        run: |
          for i in $(seq 1 30); do
            if curl -s -o /dev/null -w "%{http_code}" http://localhost:8889/wp-login.php | grep -q "200"; then
              echo "WordPress is ready"
              break
            fi
            echo "Waiting for WordPress... (attempt $i)"
            sleep 2
          done

      - name: Run E2E tests
        run: npm run test:e2e

      - name: Upload Playwright report on failure
        if: ${{ !cancelled() }}
        uses: actions/upload-artifact@v7
        with:
          name: playwright-report
          path: |
            playwright-report/
            tests/e2e/artifacts/test-results/
          retention-days: 7
          if-no-files-found: ignore

      - name: Stop WordPress environment
        if: always()
        run: npx wp-env stop
```

Key decisions:
- **Separate workflow file** (TOOL-03): `e2e.yml` is a sibling of `phpunit.yml`, not part of it. The two CI pipelines have zero shared jobs, steps, or environment variables.
- **ubuntu-24.04**: Matches the existing phpunit.yml runner. Docker 28.0.4 and Docker Compose 2.38.2 are pre-installed.
- **actions/setup-node@v4 with node-version-file**: Reads `.nvmrc` for Node version. Uses npm cache to speed up `npm ci`.
- **npx playwright install --with-deps chromium**: Chromium only (~300MB, not full ~800MB). `--with-deps` installs system-level dependencies (libnss3, etc.). Per Playwright docs, browser binary caching is NOT recommended — download each time.
- **npx wp-env start**: Starts the Docker-based WordPress environment. Uses `.wp-env.json` from repo root.
- **Wait for WordPress loop**: Polls `wp-login.php` up to 60 seconds. Addresses Pitfall 5 (cold-start latency). wp-env may report success before WordPress is fully initialized.
- **npm run test:e2e**: Uses the npm script from package.json, which points to `playwright test --config tests/e2e/playwright.config.ts`.
- **Artifact upload on !cancelled()**: Uploads both the HTML report and test-results (screenshots, traces) even on failure. `if-no-files-found: ignore` prevents the step from failing when tests pass (no failure artifacts generated).
- **wp-env stop always**: Cleanup step runs regardless of test outcome.
- **No PHP matrix**: Browser behavior is PHP-version-agnostic. One PHP version (8.2 from .wp-env.json) is sufficient for E2E.
- **No WP version matrix yet**: Added in Phase 7 when visual regression baselines need multiple WP versions.
- **timeout-minutes: 30**: Generous but bounded. wp-env start (~90s) + Chromium install (~90s) + tests should complete well within this.
- **Triggers on push to main and PRs**: Same pattern as phpunit.yml. No nightly schedule for E2E yet — add when the test suite is larger.

Do NOT add `e2e-tests` to the `needs` array of `notify-on-failure` in `phpunit.yml`. That job lives in a different workflow file and cross-workflow dependencies are not supported in GitHub Actions. E2E failure notification can be added as a separate job in e2e.yml later.

Rationale for each entry:
- `tests/e2e/artifacts/` — storageState files (contain auth cookies), screenshots, traces, test-results. All ephemeral, generated per run.

- `test-results/` — Playwright's default output directory (in case it writes here as well as under artifacts/).
- `.wp-env/` — wp-env's local state directory (Docker volume references, hash files).

The existing `.gitignore` already has `node_modules/` at the top — no need to add it again.
